# エンジニア数が3倍になる過程でのエンジニア組織の段階的育成アプローチ

技術負債解消の取り組みが進まない

## なぜ出来ない？

 - 優先度が上がらない
   関係者間ですりあっていない -> 言いたいことが相手に十分伝わっていない -> 相手に伝わる言葉で伝えていない

サポート期限が切れるのでバージョンアップすべき
 という伝え方だと
 「売上貢献度的にもっと優先度の高い施策がある」には擦り合わない

 -> サポートが切れることで脆弱性の攻撃が発生し、損害や機会損失が起こるリスクがある
    というように相手の興味領域置き換えて伝える

## 技術施策の推進

中規模施策以上は部署横断する必要がある
